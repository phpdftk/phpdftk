phpdftk's `html-to-pdf` package currently ships at **95.72% in-scope pass on the HTML Web Platform Tests subset** (559 / 584). The remaining 25 in-scope failures sit alongside the 277 pending-substrate clusters documented below; the same `wpt run --filter='html/**'` invocation reproduces the current breakdown.

This page is the audit trail. The shape mirrors [SVG WPT substrate](/rendering/svg-wpt-substrate/) and [MathML WPT substrate](/rendering/mathml-wpt-substrate/) — same four-bucket classification, applied to the HTML corpus.

## Pending substrate (what's deferred)

Pending-substrate fixtures fall into nine substrate clusters. Each is recoverable given the noted work; none reflects a defect in shipped functionality.

### Top layer + `::backdrop` (phase 4G)

Dialogs (`<dialog>` open) and popovers (`[popover]` shown) escape the document's normal stacking contexts into the *top layer* — they render above all other content regardless of ancestor `transform` / `filter` / `contain` / `will-change` / `overflow: clip`. The `::backdrop` pseudo-element generates a paint box behind each top-layer element. Both need a separate paint-order substrate that doesn't exist yet.

**Globs**:
- `html/semantics/interactive-elements/the-dialog-element/top-layer-*`
- `html/semantics/interactive-elements/the-dialog-element/backdrop-*`
- `html/semantics/interactive-elements/the-dialog-element/fixed-position-child-with-*`
- `html/semantics/interactive-elements/the-dialog-element/top-layer-containing-block`
- `html/semantics/interactive-elements/the-dialog-element/top-layer-display-none`
- `html/semantics/interactive-elements/the-dialog-element/top-layer-nesting`
- `html/semantics/interactive-elements/the-dialog-element/removed-element-is-removed-from-top-layer`
- `html/semantics/interactive-elements/the-dialog-element/dialog-popover-overlay`
- `html/semantics/popovers/popover-*-backdrop`

### Fieldset anonymous box + legend cutout (phase 4G)

HTML 5 §15.5.13 makes `<fieldset>` produce an anonymous *contents-box* inside its border-box, with `<legend>` rendered in the border-box itself (cutting a gap in the top border). Layout properties on the fieldset apply to the border-box but `overflow`/`scroll`/`containing-block` apply to the anonymous contents-box — that split isn't in our box generator yet.

**Globs**:
- `html/rendering/non-replaced-elements/the-fieldset-and-legend-elements/fieldset-containing-block`
- `.../fieldset-painting-order`
- `.../fieldset-border-radius-with-alpha`
- `.../fieldset-border-gap-position-relative`

- `.../empty-scrollable`
- `.../legend-position-relative`

### Customizable `<select>` pseudos (phase 4G)

The CSS-customizable-select feature (Chromium 2025) exposes `<select>` internals through `appearance: base-select` and a new set of pseudo-elements: `::picker(select)`, `::checkmark`, `::picker-icon`, plus `::before`/`::after` slots on the select grid. Brand-new web platform feature; substrate plan TBD.

**Globs**:
- `html/semantics/forms/the-select-element/customizable-select/**`

### `interestfor` HTML attribute (phase 4G)

The Open UI *interest-invokers* proposal (2025) adds an `interestfor` attribute plus `::interest-button` pseudo-element. Tentative; depends on the same top-layer + anchor-positioning substrate as popover.

**Globs**:

## Why this position is right for v1

Same three considerations as the [MathML position](/rendering/mathml-wpt-substrate/#why-this-position-is-right-for-v1):

- **Static rendering is our scope**. Where the WPT corpus exercises runtime behaviour (canvas draw calls, top-layer transitions, dynamic content insertion), marking those out-of-scope or pending-substrate is honest.
- **Real-world content uses the in-scope subset**. Print-targeted HTML — reports, invoices, manuscripts, dashboards — sits squarely in the in-scope subset and renders cleanly.
- **The substrate items are well-scoped follow-ups**. None is gated on a third party; each is a discrete v2 substrate plan.

## What changes in v2

The pending-substrate clusters above are the v2 backlog. By fixture count, the largest leverage items:

1. **Top layer + `::backdrop` substrate** (~50 fixtures across dialog + popover). One substrate change unlocks both elements.
2. **Customizable-select pseudos** (`::picker`, `::checkmark`, etc.) (~150 fixtures including currently-skipped). Brand-new CSS feature with a discrete substrate plan.
3. **Fieldset anonymous contents-box + legend cutout** (~11 + currently-skipped). Self-contained box-model patch.
4. **HTML 4 presentational hints** (`hr`/`br`/`table-border`) (~7 fixtures). Small `applyPresentationalAttributes` extensions; UA stylesheet alignment for `<hr>` covered alongside.
5. **Resource loader substrate** (~25 fixtures, shared with SVG/MathML).
6. **Form-widget rendering specifics** (~8 fixtures). Anonymous content boxes on `<button>`, intrinsic sizing on checkbox/radio.

None of these is required for shipping a static-PDF HTML renderer. Each is a deliberate v2 substrate item.

## One real fix in this sweep

The HTML WPT drive surfaced one real renderer bug, not a substrate gap:

- `<link rel="alternate stylesheet">` was being applied by default. Per HTML 5 §4.6.7.10, alternate stylesheets are user-opt-in; persistent and preferred sheets apply by default but alternate sheets do not. `Renderer::loadLinkedStylesheet` now drops `<link rel>` token lists that include `alternate`. Fixes `html/semantics/links/linktypes/alternate-css`.
